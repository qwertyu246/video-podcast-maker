# Topic Definition — OpenClaw Intro

**Video name:** openclaw-intro
**Target:** Engineering team internal introduction
**Platform:** Internal demo / knowledge sharing
**Language:** 繁體中文

## Topic

介紹 OpenClaw：什麼是 Agent 編排 orchestration 系統，以及它如何解決單一 Agent 的瓶頸問題。

## Target Audience

工程團隊成員，有基本軟體開發經驗，熟悉 Agent / LLM 概念但不需要專家。

## Video Style

知識型短片，motion graphics 風格，重視「動態感」與「影片感」，不做靜態投影片。

## Content Scope

- OpenClaw 是什麼（定位：非聊天機器人，是編排架構）
- 為什麼需要它（單一 Agent 的瓶頸問題）
- 核心概念（Agent、Route、Binding、Tool/Script、Agent Team）
- 實際運作流程（入口 → Binding → Route → Agent → 結果回報）
- 使用情境（Discord 分流、任務分工、自動化測試、文件整理、Infra）
- 與單一 Coding Agent 的差異
- 總結價值

## Tone

清楚、具體、工程師視角、不太行銷。

## Duration

約 3 分鐘（2:50 ~ 3:20 區間）。

## Section Structure

| Section | Content |
|---------|---------|
| hero | OpenClaw 登場，定義它是什麼 |
| problem | 單一 Agent 瓶頸 |
| concepts | 五大核心概念 |
| routing | 實際運作流程 |
| use_cases | 五種使用情境 |
| comparison | 與單一 Coding Agent 差異 |
| summary | 價值總結 |

## Visual Direction

Motion graphics 風格，每個 section 有明確的視覺化空間：
- Hero：OpenClaw 中央 node + 多 Agent 網絡線條
- Problem：單一節點被任務淹沒
- Concepts：節點 + 卡片 + 連線逐一展開
- Routing：入口 → Route Layer → 分流 → Agent → 回報
- Use cases：各情境的 flow diagram
- Comparison：左右對比（單一 vs 編排）
- Summary：所有概念收斂全景圖

## English Terms (中文 + 英文)

首次出現時附說明：
- Agent 編排 orchestration
- 路由 Route
- 綁定 Binding
- 工具 Tool
- 腳本 Script
- Agent 團隊 Agent Team
- 單一 Coding Agent
